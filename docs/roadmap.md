# Youji Development Roadmap

## Completed
- Phase 0: Core save system (auth, URL save, screenshot, manual entry, Google Places)
- Phase 1: Horizon experience (uniform cards, icons, toggle, geographic grouping, simplified filters)
- Phase 2: Rapid capture (quick add, bulk paste, background resolution)
- Phase 3: Trip creation and planning (destination model, trip overview, destination detail pages, route grouping, suggestions, notes, date picker)
- Phase 4: Collaboration (companions, comments, votes)
- Phase 5: Sharing (share links, copy/fork trips)
- Phase 6: Analytics (event tracking)
- Nomenclature rename (Horizon, Someday/Planning/Upcoming, etc.)
- Bilingual place names
- Google Places data auto-population
- Deployment to Vercel

## Current — Polish & Mobile Optimization
- Navigation stack fixes
- Trip overview redesign (hero image, compact icons, date-derived from destinations)
- Destination detail page (full-page editing environment)
- Checklist-style trip-wide notes
- Swipe-to-delete gestures
- Long-press drag-and-drop
- Context-aware + button (capture only: Save a link, Photo, Add places)
- Global search (4th nav tab)
- Visual fixes (image loading, carousel alignment, input sizing)
- Mobile UX refinements
- Airbnb-style calendar range date picker

## Next — Booking Information & Transport
- Accommodation fields per destination (hotel name, confirmation, dates, link)
- Transport fields between destinations (flight/train details, times, confirmation)
- Transport visual connectors (illustrated dotted pathway with logistics)
- Google Maps export

## Next — Trip Diary / Completion Layer
- Post-trip rating/tagging of activities
- Trip summary generation
- Photo attachment to completed activities
- This is a prerequisite for both the social intelligence layer AND the marketplace

## Next — Launch Prep & User Growth
- Onboarding flow (guided first save, wow moment)
- Empty state polish
- Mobile responsiveness audit
- Performance optimization
- Get product into friends' hands, validate core loop
- Goal: build a base of users creating and completing quality trips

## Next — Social Intelligence Phase 1
- Privacy model implementation (metadata vs content visibility levels)
- Horizon "Friends' Activity" toggle
- Planning-the-same-region matching
- Trip overlap detection with notifications
- Expert friend routing on destination pages
- City-level friend aggregation from completed trips
- "Steal this" from friends' completed trips

## Later — Marketplace (after user base and trip supply exist)
- Marketplace data model (listings, pricing, creator profiles)
- Creator flow: mark trip as for sale, set price, add description, embed external content
- Buyer flow: browse listings, preview trip, purchase, fork into account
- Linked fork updates (creator changes propagate as pull-based notifications)
